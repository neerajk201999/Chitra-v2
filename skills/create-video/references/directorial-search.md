# Still-first directorial search

Use this route when the user delegates a meaningful creative choice and more
than one governing idea could truthfully solve the locked Intake. Its purpose is
to reject a weak Direction before expensive motion, not to generate options for
their own sake.

## Trigger and skip rules

Run a search for an open, ambiguous, unfamiliar, or high-stakes brief when the
governing idea, emotional turn, hero composition, or narrative mechanism is not
already approved. Use two candidates when the real choice is binary; use three
by default; use four only when all six pairwise distinctions remain material.

Skip with a written reason when:

- the user supplied and approved an exact Direction;
- the job is a faithful reference reconstruction and the remaining uncertainty
  is execution rather than concept;
- a prior Direction Selection receipt binds the same Intake;
- rights, capability fit, or missing product truth blocks every candidate.

## Authoring contract

1. Lock Intake first. Candidate Directions share objective, register, sources,
   brand constraints, and must-have capability truth.
2. Make each candidate complete enough to judge: distinct governing idea,
   narrative mechanism, hero composition, motion premise, tradeoff, and likely
   failure. Synonyms, palette swaps, and alternate copy on one layout are one
   candidate, not several.
3. Declare every pairwise difference in idea, story, and image. If the
   distinction cannot survive this comparison, merge or delete the candidate.
4. Build only one to three diagnostic probe Score scenes per candidate, using
   the same ordered roles (`opening`, `hero`, `resolution`) and the same question
   for each comparable role. Do not build parallel Storyboards or full films.
5. Bind the draft with `chitra direction-search-lock`, then render with
   `chitra direction-probes`. A hard capability/static/render defect blocks the
   packet; high-priority style flags stay visible but do not kill an authored
   option.

## Blind review contract

The reviewer receives only `blind-packet.json`, `blind-contact-sheet.png`, and
the identity-free probe stills. The authoring agent, candidate Direction files,
private manifest, candidate names, and mapping remain hidden until the decision
is recorded. If one model must perform both roles, use a fresh isolated context
and declare `kind: model`; this is procedural blinding, not proof of independence.

Record the first read before scoring. Assess governing-idea legibility,
emotional charge, hierarchy, brand specificity, narrative potential, and motion
potential separately. Rank all candidates, explain every loser, retain useful
salvage without averaging candidates together, and state uncertainty. Scores
organize evidence; never sum them into an automatic taste winner.

`chitra direction-select` re-hashes the evidence and resolves the blind winner
through the private manifest. Its receipt authorizes that exact Direction to
advance to Storyboard. Changed evidence or a pixel-near-identical rendered
search blocks selection. The receipt proves process integrity, not professional
taste, originality, or genuine reviewer blindness.
