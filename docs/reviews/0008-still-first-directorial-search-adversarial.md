# Review 0008 — still-first directorial search adversarial review

**Date:** 2026-07-18

**Scope:** ADR-0036 schemas, lock, probe generation/cache, blind packet,
selection validation, receipt, CLI, benchmark, and agent route.

## Verdict

The 0.1 transaction is fit to ship as a bounded process and integrity primitive.
It does not establish professional taste, genuine blindness, originality, or
motion/audio quality. Those claims remain blocked on independent real outcomes.

## Findings resolved before shipment

1. **Public packet leaked private scene and Direction beat IDs.** Public probe
   artifacts now contain only a blind evidence ID, role, question, path, hash,
   and byte count.
2. **Candidate-authored questions could reveal identity or make comparisons
   incomparable.** Every candidate now uses identical questions for each shared
   ordered role; candidate/Direction identities are rejected inside questions.
3. **Changing evidence after review could still resolve a winner.** Receipt
   creation now requires the evidence directory and immediately re-hashes the
   packet, contact sheet, and every still.
4. **Score JSON was bound but referenced asset bytes could drift.** Search lock
   records every resolved render input and verification re-hashes it.
5. **A digest-valid private manifest could omit pair diagnostics, cite foreign
   blind IDs, or duplicate evidence IDs.** Manifest refinement now enforces
   unique private/blind/evidence IDs, identical probe roles/questions, complete
   pair×role coverage, and known-candidate duplicate declarations.
6. **The public packet and private manifest were independently hash-valid but
   not mutually checked.** Cache/receipt verification now compares search,
   candidate inventory, and every public probe descriptor across both views.

## Adversarial cases exercised

- incomplete pairwise Direction distinctions;
- mismatched probe roles/questions and identity-bearing questions;
- unknown scenes/beats, out-of-range probe offsets, register/dimension drift;
- P1 static or rendered findings;
- Direction, Score, asset, manifest, packet, contact-sheet, and still mutation;
- cache directory and project-path containment;
- duplicate candidate, Direction, blind, evidence, ranking, and review entries;
- incomplete assessment/rejection/ranking and foreign evidence ownership;
- exact cache reuse and CLI/library receipt agreement;
- two visually identical rendered candidates blocked at RGB MAE ≤ 0.01.

## Open limits

- Text distinctions can be paraphrased or gamed; semantic originality is judged,
  not deterministically proven.
- RGB MAE is a duplicate alarm only. It misses semantically identical images
  with changed palette/layout and must never rank creative quality.
- Blindness declarations bind reviewer claims but cannot prove behavior. A model
  in a fresh context is procedurally isolated, not independently expert.
- One to three still probes cannot prove motion design, timing, edit, sound, or
  the complete narrative. The winner still requires Storyboard, watched preview,
  Creative Review, and verified release.
- Digest receipts detect mutation, not malicious re-authoring of every local
  artifact; Chitra's local project files are not a cryptographic trust authority.
- Browser pixels are only evidenced on the current machine. Cross-platform
  rendered equivalence remains unproven.
- No independent study shows that this search chooses professional work or wins
  against Remotion, HyperFrames, EditFrame, or video-use workflows.

## Cost boundary

The schema caps search at four candidates and three stills each. Agents must not
create full Storyboards or films for losing candidates, load whole references,
or average dimension scores into an automatic winner. The search is skipped for
an exact approved Direction or reconstruction target, with the reason recorded.
