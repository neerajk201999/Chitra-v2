# Security Policy

## Supported version

Security fixes target the latest published `chitra-video` release and current
`main`. Older preview releases may require upgrading.

## Report a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's
**Security → Report a vulnerability** flow for this repository. Include the
affected version, reproduction steps, impact, and any suggested mitigation.

Chitra renders local files and may capture user-supplied URLs. Reports involving
path traversal, unsafe browser behavior, provenance bypass, arbitrary code
execution, or release-receipt integrity are especially useful.

The project will acknowledge a report, investigate it privately, and publish a
fix and advisory when appropriate. Please avoid accessing data you do not own
while testing.
